Ogni Branch sarà suddiviso in:

Features = Creazione nuovo elemento
BugFix = Correzione errori
Develop = Sviluppo completo di una Feature

ES:

Features-aggiunta-db/...

Ogni merge al master Dovrà essere prima testato e in seguito dovrà essere eseguita una merge-request per controllare eventuali conflitti con gli altri branch o col master
